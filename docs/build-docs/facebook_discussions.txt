When it comes to the Contextual Vectors - we order the headings depending on the context.
* The first rule is staying on the same subject as much as possible.
* If you're talking about "how to clean a pool" for instance, and in the second question, if you ask - "what is chloride?" - it doesn't work in a proper way like that.
* You should always stay in the specific Predicate.
* You should continue to talk about "cleaning the pool" and NOT the definition of the specific thing ( like "chloride").
* Usually, when people start to create certain type of web documents, they start to talk about multiple contexts at the same time. 
For instance "how to perform deadlift in a proper form?". 
It's NOT about - "what are the types of deadlifts". It's just about "how to perform deadlift with a proper form".
* Which means - you need to have a instruction list at the top.
* If I have a list at the top, I usually add one bridge - a kind of H2. 
Then I start to ask a question - "how to perform a deadlift as a beginner, or as a female or as a man, or maybe with some diseases etc",
or "from powerlifter to body-builder" (- some of the types that you will be implementing might be changing).
* So we are actually distributing the "deadlift types" still inside "performing it in a proper form" 
which means that we don't actually dilute the context - still we are staying in the same subject as much as possible.
* But we continue to mention the "types of deadlifts" or the "types of the athletes", like, the body-builder. 
Once we start to talk about that, then we start to use the internal links.
* Usually, I don't link from the top of the page. I link from the middle or MOSTLY from the bottom section.
* Because in the bottom headings, most of the time, I have 2 Main Entities 
* One of them is the Subject at the H1.
* The second one is the Subject I will be having in the targeted pages - internally linked targeted pages.
* If I am able to distribute them like that you will be justifying the connections as much as possible.
* If you're able to implement these above 3 or 4 simple things - it will help you to protect your context.
