## Overview

This document defines the user experience and interface specifications for the multi-tenancy, RBAC, and billing system.

### Design Principles

1. **Invisible for solo users** - Organization concepts hidden until needed
2. **Clear context** - Color coding ensures users always know where they are
3. **Progressive disclosure** - Complexity revealed only when relevant
4. **Minimal friction** - Inline actions, no unnecessary wizards

### Color Assignment

- **Auto-assigned on creation** from curated palette: `#7C3AED` (purple), `#0D9488` (teal), `#EA580C` (orange), `#2563EB` (blue), `#DC2626` (red), `#059669` (green)

- Personal workspaces default to neutral gray until customized

### Solo User Experience

- Users with **only one organization** see no switcher, no org-related UI
- Color accents still apply (subtle branding)
- Org features appear only when user creates second org or receives invitation
